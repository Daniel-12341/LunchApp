**Step 2: Create browser client** (`src/utils/supabase/client.ts`):

- Uses `process.env.NEXT_PUBLIC_SUPABASE_URL!` and `process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!`

**Step 3: Create server client** (`src/utils/supabase/server.ts`):
- Export an async `createClient()` function using `createServerClient` from `@supabase/ssr`
- Uses `cookies()` from `next/headers` with `getAll`/`setAll` pattern
- Wrap `setAll` in try/catch (Server Components can't write cookies)
- Follow exact pattern from RESEARCH.md Pattern 2

**Step 4: Create middleware** (`src/middleware.ts` — MUST be in src/ not root):
- Create Supabase server client with cookie pass-through (getAll from request.cookies, setAll to both request and response cookies)
- Call `supabase.auth.getUser()` (NEVER getSession()) to refresh session
- Route protection logic:
  - If NOT authenticated AND pathname is not `/` → redirect to `/`
  - If authenticated AND pathname is `/` → redirect to `/home` (so logged-in users skip login page)
- Matcher config: exclude static files, images, favicon
- Do NOT add admin role check in middleware — that happens in the admin page itself (simpler, avoids extra DB query on every request)

IMPORTANT: Use `getUser()` not `getSession()` per RESEARCH.md anti-patterns. Place file at `src/middleware.ts` not project root.
