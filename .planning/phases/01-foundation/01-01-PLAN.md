  <action>
Create three SQL migration files:

**Migration 1 — Profiles table** (`20260315000001_create_profiles.sql`):
- `id uuid primary key default gen_random_uuid()` — plain UUID, NO foreign key to auth.users (because org members have no auth accounts; see RESEARCH.md open question 1)
- `auth_user_id uuid references auth.users(id) on delete cascade` — nullable, only populated for the 2 auth-linked profiles (user and admin)
- `username text not null`
- `role text not null default 'user' check (role in ('user', 'admin'))`
- `created_at timestamptz not null default now()`
- Enable RLS on profiles
- Policy: "Profiles are viewable by authenticated users" — SELECT for `authenticated` using `(true)`
- Policy: "Users can update own profile" — UPDATE for `authenticated` using `(select auth.uid()) = auth_user_id`

**Migration 2 — Orders table** (`20260315000002_create_orders.sql`):
- `id uuid primary key default gen_random_uuid()`
- `user_id uuid not null` — references the auth user who placed the order (shared user account)
- `selected_name text not null` — the org member name chosen in Phase 2
- `meal_category text not null`
- `meal_name text not null`
- `price numeric(6,2) not null`
- `customisation text` — nullable
- `week_number integer not null`
- `year integer not null`
- `archived boolean not null default false`
- `created_at timestamptz not null default now()`
- Enable RLS on orders

**Migration 3 — RLS Policies for Orders** (`20260315000003_rls_policies.sql`):
- SELECT policy: authenticated users can read orders where `(select auth.uid()) = user_id` OR the user's profile role is 'admin' (subquery: `select role from public.profiles where auth_user_id = (select auth.uid())`)
- INSERT policy: authenticated users can insert where `(select auth.uid()) = user_id`
- UPDATE policy (user): authenticated users can update where `(select auth.uid()) = user_id`
- UPDATE policy (admin): authenticated users can update where profile role is 'admin' (subquery on profiles)

IMPORTANT: The admin role check in RLS policies must join on `auth_user_id` (not `id`) since that is the column linking to auth.users.
