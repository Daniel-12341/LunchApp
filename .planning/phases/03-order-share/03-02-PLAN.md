  <action>
Modify `OrderPageClient.tsx` to handle `step === 'confirm'` and `step === 'success'` states. The component already has `step` state from Plan 01 and sets it to `'confirm'` when the submit button is tapped.

**Add import:**
```typescript
import confetti from 'canvas-confetti'
import { saveOrder } from '@/utils/orderActions'
```

**Add state:**
- `isSubmitting: boolean` (loading guard for save)
- `submitError: string | null` (error display)

**Confirmation screen (when `step === 'confirm'`):**
Replace the selection UI with a centered review card showing:
- Header: "Review Your Order" or similar
- Selected meal name (large text)
- Category emoji + category name
- Price: "R{price}"
- Special requests (if any), displayed in a muted section
- Two buttons at bottom:
  - "Go Back" — secondary/outline style, sets `step` back to `'select'`
  - "Confirm Order" — primary emerald button, calls `handleConfirm()`
- While submitting: "Confirm Order" button shows loading spinner, both buttons disabled

**handleConfirm function:**
1. Set `isSubmitting = true`, clear `submitError`
2. Derive the selected meal data from `MENU[selectedItem.categoryIndex].items[selectedItem.itemIndex]`
3. Determine price: if `useAltPrice` and item has `priceAlt`, use `priceAlt`; otherwise use `price`
4. Call `saveOrder({ userId, selectedName: name, mealCategory: MENU[selectedItem.categoryIndex].name, mealName: item.name, price, customisation: specialRequests || undefined })`
5. If error returned: set `submitError`, set `isSubmitting = false`, stay on confirm screen
6. If success: trigger confetti, set `step` to `'success'`

**Confetti trigger (on success):**
```typescript
confetti({
  particleCount: 120,
  spread: 80,
  origin: { y: 0.6 },
  colors: ['#10b981', '#34d399', '#6ee7b7', '#fbbf24', '#f59e0b'],
})
```
Fire immediately after setting step to success (before render).

**Success screen (when `step === 'success'`):**
Full-page celebration overlay:
- Confetti raining down (already triggered above, canvas-confetti handles the visual)
- Centered modal/card with:
  - Large checkmark or party emoji at top
  - "Order Placed!" heading (large, bold, emerald)
  - Order summary: meal name, category, price, special requests (if any)
  - "Done" button — resets state: `setStep('select')`, `setSelectedItem(null)`, `setSpecialRequests('')`, `setSubmitError(null)`

**Error handling:**
If `submitError` is set on the confirmation screen, show an inline error message in red above the buttons: "Something went wrong: {error}. Please try again."

**Styling notes:**
- Confirmation card: white bg, rounded-2xl, shadow-lg, centered on page with same gradient background
- Success modal: similar styling but with celebratory colours (emerald accents, maybe a subtle green glow)
- Transitions between steps can be simple conditional rendering (no animation library needed)
- Mobile-friendly: cards should be full-width on small screens with padding
