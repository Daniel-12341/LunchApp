Create `src/utils/adminActions.ts` with 'use server' directive containing two server actions:

1. `getWeeklyOrders()` — copies the `getISOWeekNumber` function from `orderActions.ts` (same exact implementation to avoid week mismatch). Queries orders table filtered by current week_number + year + archived=false, ordered by created_at ascending. Returns `{ data: Order[], error?: string }` where Order type is `{ id: string, selected_name: string, meal_category: string, meal_name: string, price: number, customisation: string | null, created_at: string }`.

2. `archiveWeek(weekNumber: number, year: number)` — bulk UPDATE orders SET archived=true WHERE week_number=weekNumber AND year=year AND archived=false. Returns `{ count: number, error?: string }` where count is the number of archived rows. Use the server Supabase client (`createClient` from `@/utils/supabase/server`).

- Update auth.identities email and identity_data->>'email' to 'username123@lunchapp.com'
- Update profiles username from 'admin' to 'username123'

Note: The app login form constructs email as `${username}@lunchapp.com`, so username123 maps to username123@lunchapp.com.
